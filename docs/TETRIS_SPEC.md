# Retris 仕様書 — テトリス公式ガイドライン準拠 (GBA 向け)

本書はテトリス公式ガイドライン (Tetris Guideline, 2009 年版準拠) に基づく、GBA (59.73fps 固定・フレームベース) 上で動作するテトリスの実装仕様である。実装者は本書のみを参照してテストとコードを書けることを目標とし、すべての数値・座標を明記する。

- 対象フレームレート: **59.73 fps** (GBA 実機は 59.7275 Hz。本書では 59.73 で統一して換算する)
- 時間はすべて「GBA フレーム数」を正とする。秒表記は参考値。

---

## 0. 用語と略記

| 用語 | 意味 |
|---|---|
| ミノ / ピース | テトロミノ (I, O, T, S, Z, J, L の 7 種) |
| セル | フィールドの 1 マス |
| 回転状態 | `0` = スポーン向き, `R` = 右 (時計回り) 90°, `2` = 180°, `L` = 左 (反時計回り) 90° |
| CW / CCW | 時計回り回転 / 反時計回り回転 |
| 接地 | ミノが現在位置から 1 行下に移動できない状態 |
| G | 重力単位。1G = 1 行/フレーム |

## 1. フィールド

- 幅 **10 列 × 高さ 40 行**。うち下 20 行が**可視領域**、上 20 行が**バッファ領域**(不可視)。
- 実用上バッファは 20 行も使われない (キックによる上方移動は最大 +2) が、配列は 10×40 で確保する。

### 1.1 座標系 (本書全体で一貫)

- **x**: 左端 `x=0`、右端 `x=9`。右方向が正。
- **y**: 最下行 `y=0`、最上行 `y=39`。**上方向が正**。
  - 可視領域: `y = 0..19`
  - バッファ領域: `y = 20..39`
  - ガイドライン文書の「第 n 行 (1 始まり・下から)」は本書の `y = n-1` に対応する (例: 第 21 行 = `y=20`)。
- セルの有効判定: `0 <= x <= 9` かつ `0 <= y <= 39` かつ `field[y][x]` が空。フィールド外 (左右壁・床) は「埋まっている」扱い。
- **描画時の変換**: GBA の画面座標は y が下向きなので、可視セル `(x, y)` (y=0..19) をタイル座標 `(x, 19 - y)` に変換して描画する。ロジック層は本座標系のみを使うこと。

出典: https://tetris.wiki/Playfield , https://tetris.wiki/Tetris_Guideline

## 2. テトロミノ定義

### 2.1 バウンディングボックスと回転中心

- **J, L, S, T, Z**: 3×3 ボックス。回転中心はボックス中央セル `(1,1)`。
- **I**: 4×4 ボックス。回転中心はボックス中央 (セル間の点)。
- **O**: 2×2 ボックス。回転しても形・位置が変わらない (§4.4)。

ミノの状態は「ボックス左下隅のフィールド座標 `(bx, by)` + 回転状態」で表す。ボックス内セル座標 `(cx, cy)` (左下原点・y 上向き、フィールド座標と同じ向き) を用い、実セル位置は `(bx + cx, by + cy)`。回転はボックス内のセル配置をテーブルで差し替えるだけで実装できる (以下の表がそのテーブル)。

### 2.2 全回転状態のセル座標 (SRS 基準)

ボックス内座標 `(cx, cy)`、左下原点。

| ミノ | 色 | RGB888 (参考) | GBA RGB555 (推奨値) |

## 3. スポーン

- 全ミノ**状態 0** (水平・フラット面下向き) でスポーンする。
- スポーン行: ガイドラインの第 21〜22 行、すなわち本座標系の **y=20〜21** (可視領域のすぐ上)。
- 水平位置: I と O は中央、3 幅ミノ (J, L, S, T, Z) は「中央から左寄せ」= 列 x=3..5 を占有。

### 3.1 スポーン時のボックス左下座標 `(bx, by)`

| ミノ | (bx, by) | 占有セル (フィールド座標) |
|---|---|---|
| I | (3, 18) | (3,20) (4,20) (5,20) (6,20) |
| O | (4, 20) | (4,20) (5,20) (4,21) (5,21) |
| T | (3, 19) | (3,20) (4,20) (5,20) (4,21) |
| S | (3, 19) | (3,20) (4,20) (4,21) (5,21) |
| Z | (3, 19) | (4,20) (5,20) (3,21) (4,21) |
| J | (3, 19) | (3,20) (4,20) (5,20) (3,21) |
| L | (3, 19) | (3,20) (4,20) (5,20) (5,21) |

### 3.2 スポーン直後の挙動

1. 上表の位置にミノを生成する。
2. **ブロックアウト判定**: 生成位置の 4 セルのいずれかが既存ブロックと重なる場合、その場でゲームオーバー (§13)。ミノはフィールドに重ねて描画してよい (視覚上の慣例)。
3. 重なりがなければ、**生成した同一フレームで 1 行落下を試みる** (下が空いていれば `y -= 1`)。これはガイドライン規定の挙動。落下できない場合はその位置のまま接地扱いとなりロックディレイが開始する。
4. 重力アキュムレータ (§7) は 0 から開始する。

出典: https://tetris.wiki/Tetris_Guideline , https://harddrop.com/wiki/Spawn_Location

## 4. SRS (スーパーローテーションシステム)

回転入力時の処理:

1. 回転状態をテーブル (§2.2) に従って差し替える (位置 `(bx, by)` は不変)。
2. その配置で衝突判定。衝突しなければ確定 (これが「テスト 1 = (0,0)」)。
3. 衝突する場合、下表のオフセット `(dx, dy)` をテスト 2→5 の順に `(bx+dx, by+dy)` へ適用し、最初に衝突しない位置で確定。
4. 5 テストすべて失敗なら**回転は不発** (状態・位置とも変更なし。ロックディレイのリセットも発生しない)。

符号規約: **+x = 右、+y = 上** (本書の座標系そのまま。tetris.wiki の表と同一符号)。

### 4.1 J, L, S, T, Z 用キックテーブル

| 遷移 | テスト 1 | テスト 2 | テスト 3 | テスト 4 | テスト 5 |
|---|---|---|---|---|---|
| 0→R | (0, 0) | (−1, 0) | (−1, +1) | (0, −2) | (−1, −2) |
| R→0 | (0, 0) | (+1, 0) | (+1, −1) | (0, +2) | (+1, +2) |
| R→2 | (0, 0) | (+1, 0) | (+1, −1) | (0, +2) | (+1, +2) |
| 2→R | (0, 0) | (−1, 0) | (−1, +1) | (0, −2) | (−1, −2) |
| 2→L | (0, 0) | (+1, 0) | (+1, +1) | (0, −2) | (+1, −2) |
| L→2 | (0, 0) | (−1, 0) | (−1, −1) | (0, +2) | (−1, +2) |
| L→0 | (0, 0) | (−1, 0) | (−1, −1) | (0, +2) | (−1, +2) |
| 0→L | (0, 0) | (+1, 0) | (+1, +1) | (0, −2) | (+1, −2) |

検算用の性質: 逆遷移のテーブルは符号反転 (例: R→0 の各値 = 0→R の各値 × −1)。

### 4.3 実装メモ

- T-Spin 判定 (§10) のため、**回転確定時に「使用したテスト番号 (1〜5)」を記録**すること。
- 回転成功はロックディレイのリセット対象 (§8)。不発はリセットしない。

### 4.4 O ミノ

O は回転入力を受けても**状態・位置とも変化しない** (キックも発生しない)。回転入力は成功とも失敗とも扱わず、**ロックディレイのリセットも発生させない**こと (無限回転防止)。

### 4.5 180° 回転

**不採用**。CW / CCW の 90° 回転のみ実装する。

出典: https://tetris.wiki/Super_Rotation_System , https://harddrop.com/wiki/SRS

## 5. 7 バッグランダマイザー (Random Generator)

### 5.1 アルゴリズム

1. 7 種のミノ 1 個ずつを袋 (bag) に入れる: `[I, O, T, S, Z, J, L]`。
2. Fisher–Yates で袋をシャッフルする:
   ```
   for i = 6 downto 1:
       j = rand_bounded(i + 1)   // 0 <= j <= i の一様乱数
       swap(bag[i], bag[j])
   ```
3. 袋から順に取り出してネクストキューへ供給。袋が空になったら 1. に戻って新しい袋を作る。
- 同一ミノは最大でも 2 連続 (袋境界のみ)、同一ミノの出現間隔は最大 12 手。
- ネクスト 5 個表示 (§6) のため、キュー長は常に 6 以上 (操作中ミノ+5) を保つよう袋を補充する。

### 5.2 擬似乱数 (GBA 向け決定的 PRNG)

GBA に OS 乱数はないため **xorshift32** を使用する:

```c
static u32 rng_state;          // 0 は禁止 (初期化時に保証する)

static u32 xorshift32(void) {
    u32 x = rng_state;
    x ^= x << 13;
    x ^= x >> 17;
    x ^= x << 5;
    return rng_state = x;
}

// 0 <= 戻り値 < n (n <= 256 程度なら偏りは無視できる)
static u32 rand_bounded(u32 n) {
    return ((xorshift32() >> 16) * n) >> 16;
}
```

### 5.3 シード戦略

- タイトル画面表示中、VBlank ごとにフレームカウンタ `frame_count` (u32) をインクリメントする。
- ゲーム開始ボタン押下時に `rng_state = (frame_count * 2654435761u) | 1;` とする (Knuth 乗算ハッシュで拡散、`|1` で 0 を回避)。
- 人間の入力タイミング (フレーム精度) がエントロピー源となる。リプレイ・デバッグ用途にはシード値を固定入力できる仕組みを設けてよい。

出典: https://tetris.wiki/Random_Generator , https://tetris.wiki/Tetris_Guideline

## 6. ネクストとホールド

### 6.1 ネクスト

- **5 個**を先読み表示する (キュー先頭が次にスポーンするミノ)。

### 6.2 ホールド

- ホールド枠は **1 個**。
- ホールドボタン押下時:
  - ホールド枠が空: 操作中ミノをホールド枠に格納し、ネクストキュー先頭のミノをスポーンさせる。
  - ホールド枠にミノあり: 操作中ミノとホールド枠のミノを交換し、ホールドされていたミノを §3.1 のスポーン位置・状態 0 でスポーンさせる (スポーン直後の 1 行落下 §3.2-3 も行う)。
- **連続使用不可**: ホールドを使うと `hold_used = true` になり、次のミノが**ロックされるまで**再ホールドは無効 (入力を無視)。ロック時に `hold_used = false` に戻す。
- ホールドから出したミノのスポーンでもブロックアウト判定 (§3.2-2) を行う。
- IHS (Initial Hold System) は任意 (§12.4)。

出典: https://tetris.wiki/Tetris_Guideline , https://harddrop.com/wiki/Hold_piece

## 7. 重力 (落下速度)

### 7.1 公式

レベル L における落下時間 (秒/行):

```
t(L) = (0.8 − (L − 1) × 0.007) ^ (L − 1)
```

(Tetris Worlds 由来、現行ガイドラインのマラソン標準カーブ)

### 7.2 レベル 1〜20 数表 (59.73fps 換算)

### 7.3 ソフトドロップ

- 下キー保持中は重力を **20 倍**にする: `G_soft = G × 20` (Q16 のままオーバーフローなし。u32 で安全)。
- 例: レベル 1 で 59.73/20 = 約 3 フレーム/行。
- ソフトドロップによる落下 1 セルごとに 1 点 (§9.4)。接地中に下キーを押してもスコアは入らない。

### 7.4 ハードドロップ

- 上入力 (§12) で、ミノを**即時に接地位置まで落下させ、その場で即ロック**する (ロックディレイなし・移動リセット不可)。
- 落下 1 セルごとに 2 点 (§9.4)。落下 0 セル (すでに接地) でもロックは行う。

## 8. ロックディレイ (Extended Placement Lockdown / move reset 方式)

| パラメータ | 値 |
|---|---|
| ロックディレイ | 0.5 秒 = **30 フレーム** (0.5 × 59.73 = 29.87 → 切り上げ 30) |
| リセット回数上限 | **15 回** |

### 8.1 状態変数

- `lock_timer` : 残りフレーム数 (接地中のみ減算)
- `reset_count` : 使用済みリセット回数 (ミノごとに 0 から)
- `lowest_y` : このミノが到達した最低 (最小) の `by` (ミノごとにスポーン時の `by` で初期化)

### 8.2 規則

1. 接地 (1 行下へ移動不可) している間、毎フレーム `lock_timer` を 1 減らす。0 になった瞬間にロック (§8.3)。
2. 接地中に**移動 (左右) または回転が成功**したとき:
   - `reset_count < 15` なら `reset_count += 1` し、`lock_timer = 30` にリセット。
   - `reset_count >= 15` ならリセットしない (タイマーは減り続ける)。
3. **`reset_count >= 15` の状態で接地している場合、`lock_timer` の残りに関係なく即ロック**する (15 回使い切った後は着地即固定)。
4. 空中にいる間 (接地していない間) は `lock_timer` を減らさない。接地→空中→再接地でもタイマーは**リセットされない** (値を保持)。タイマー回復は規則 2 と 5 のみ。
5. **下降によるリセット回数回復**: 落下 (重力・ソフトドロップいずれでも) により `by < lowest_y` となったとき、`lowest_y` を更新し、`reset_count = 0`、`lock_timer = 30` に戻す。
6. 移動・回転の**失敗** (壁に当たって動けない、回転不発) はリセットしない。O ミノの回転入力もリセットしない (§4.4)。
7. ハードドロップは全規則を無視して即ロック。

### 8.3 ロック処理

ミノの 4 セルを `field` に書き込み、ロックアウト判定 (§13)、ライン消去判定 (§9.1)、`hold_used` クリア、T-Spin 判定 (§10)、スコア加算、次ミノのスポーン、と続く。

出典: https://tetris.wiki/Lock_delay , https://tetris.fandom.com/wiki/Infinity (Extended Placement / move reset), 2009 Tetris Design Guideline

## 9. ライン消去とスコア

### 9.1 ライン消去

- ロック直後、`y = 0..39` の全行を走査し、10 セルすべて埋まった行を消去。上の行を詰めて落とす (ナイーブ重力。連鎖落下なし)。
- 消去演出のためのディレイは任意 (0 フレームで即消去がデフォルト。演出を入れる場合もロジックはロック時点で確定させること)。

### 9.2 基本スコア (× レベル)

`level` は消去が発生した時点のレベル。

| アクション | 基本点 (× level) |
|---|---:|
| Single (1 列) | 100 |
| Double (2 列) | 300 |
| Triple (3 列) | 500 |
| Tetris (4 列) | 800 |
| T-Spin Mini (消去なし) | 100 |
| T-Spin Mini Single | 200 |
| T-Spin Mini Double | 400 |
| T-Spin (消去なし) | 400 |
| T-Spin Single | 800 |
| T-Spin Double | 1200 |
| T-Spin Triple | 1600 |

### 9.3 Back-to-Back (B2B)

- 「難しい消去」= **Tetris、および消去を伴う T-Spin / T-Spin Mini**。
- 難しい消去の後、間に通常消去 (Single/Double/Triple) を挟まず再び難しい消去をすると、その消去の基本点が **×1.5** (整数になる。例: Tetris 800→1200、TSD 1200→1800、TST 1600→2400、Mini TSS 200→300)。
- B2B チェーンは通常消去で切れる。**消去を伴わないミノ設置や消去なし T-Spin ではチェーンは切れない** (加点対象にもならない)。ホールドでも切れない。
- 倍率はコンボ点・ドロップ点・Perfect Clear ボーナスには掛からない。

### 9.4 コンボ

- 連続したミノ設置でライン消去が続くとコンボ。カウンタ `combo` は **−1 で初期化**し、消去ありロックで `+1`、消去なしロックで `−1` にリセット。
- `combo >= 1` のとき、追加点 `50 × combo × level` (1 回目の連続消去 = combo 1 で 50 × level)。

### 9.5 ドロップ点 (レベル倍率なし)

| アクション | 点 |
|---|---:|
| ソフトドロップ | 1 点 / セル |
| ハードドロップ | 2 点 / セル (最大 40 点 = 20 セル) |

### 9.6 Perfect Clear (全消し) ボーナス

消去後にフィールドが完全に空になった場合、**通常の消去点に加算**する (× level):

| アクション | 加算点 (× level) |
|---|---:|
| Single Perfect Clear | 800 |
| Double Perfect Clear | 1200 |
| Triple Perfect Clear | 1800 |
| Tetris Perfect Clear | 2000 |
| B2B Tetris Perfect Clear | 3200 (Tetris PC が B2B のとき、2000 の代わりに適用) |

例: Triple での全消し = 500×level (Triple) + 1800×level (PC ボーナス)。

### 9.7 加算順序 (実装規約)

ロック 1 回で加算するスコア = `ドロップ点 + floor(基本点 × B2B倍率) × level + コンボ点 + PCボーナス × level`。

出典: https://tetris.wiki/Scoring (Recent guideline 表), https://tetris.wiki/Perfect_clear

## 10. T-Spin 判定 (3 コーナールール)

### 10.1 前提条件

以下をすべて満たすときのみ T-Spin / T-Spin Mini になりうる:

1. ロックしたミノが **T** である。
2. **最後に成功した操作が回転**である (回転後に左右移動・落下 (重力落下含む) が 1 セルでも起これば無効)。実装: `last_action_rotate` フラグを回転成功時に true、左右移動成功時・1 行落下時に false にする。
3. T の 3×3 ボックスの 4 コーナーのうち **3 つ以上が埋まっている** (フィールド外=壁・床は「埋まっている」扱い)。

### 10.2 コーナーの定義

T のボックス内 4 コーナー: `A=(0,2)` 左上, `B=(2,2)` 右上, `C=(0,0)` 左下, `D=(2,0)` 右下 (ボックス内座標、フィールド座標は `(bx,by)` を加算)。

「前面」= T の凸 (尖り) が向いている側の 2 コーナー:

| 回転状態 | 尖りの向き | 前面コーナー | 背面コーナー |
|---|---|---|---|
| 0 | 上 | A (0,2), B (2,2) | C (0,0), D (2,0) |
| R | 右 | B (2,2), D (2,0) | A (0,2), C (0,0) |
| 2 | 下 | C (0,0), D (2,0) | A (0,2), B (2,2) |
| L | 左 | A (0,2), C (0,0) | B (2,2), D (2,0) |

### 10.3 判定

前提条件を満たすとき:

- **フル T-Spin**: 前面 2 コーナーが両方埋まっている、かつ背面コーナーが 1 つ以上埋まっている。
- **T-Spin Mini**: 前面コーナーが 1 つだけ埋まっていて、背面 2 コーナーが両方埋まっている。
- **キック 5 例外**: 最後の回転が**キックテスト 5 番目** (オフセット `(±1, −2)` / `(±1, +2)`、§4.1 の 5 列目) で成立した場合、Mini 条件でも**常にフル T-Spin** として扱う (いわゆる TST / Fin 型)。
- 上記いずれにも該当しなければ通常の消去 (T-Spin なし)。

スコアは §9.2 の表に従う。消去を伴う T-Spin / Mini は B2B 対象 (§9.3)。

出典: https://tetris.wiki/T-Spin , https://harddrop.com/wiki/T-Spin

## 11. レベル進行

- 累計消去ライン数 `total_lines` を数える (T-Spin でも 1 消去は 1 ライン)。
- **10 ライン消去ごとにレベル +1** (Fixed Goal): `level = start_level + total_lines / 10` (整数除算)。
- 開始レベル `start_level` はタイトル画面で **1〜15** から選択可能 (デフォルト 1)。
- レベル上限: 重力・スコア倍率とも **20 で頭打ち** (それ以上はレベル 20 の値を使い続ける)。レベル表示は増え続けてよい。
- レベルダウンはない。

出典: https://tetris.wiki/Marathon , https://tetris.wiki/Tetris_Guideline

## 12. 操作 (GBA 向け)

### 12.1 ボタン割当

| ボタン | 機能 |
|---|---|
| 十字キー左 / 右 | 左 / 右移動 (DAS/ARR 適用) |
| 十字キー下 | ソフトドロップ (押している間 20 倍重力) |
| 十字キー上 | ハードドロップ (押した瞬間のみ。押しっぱなしで連続発動しない) |
| A | 右回転 (CW) |
| B | 左回転 (CCW) |
| L または R | ホールド |
| START | ポーズ / 解除 |
| SELECT | (任意) ゲーム放棄→リザルト |

### 12.2 DAS / ARR

ガイドライン系標準値 (Puyo Puyo Tetris / Tetris Effect のデフォルト相当) を GBA フレームに換算:

| パラメータ | 値 (GBA フレーム) | 参考 |
|---|---:|---|
| DAS (初回リピートまでの遅延) | **10 フレーム** (≈167 ms) | 60fps 環境の 10F と実質同値 |
| ARR (リピート間隔) | **2 フレーム** (≈33 ms) | 同上 |

挙動:

- 押下フレームで 1 セル移動。以後押し続けると 10 フレーム後に 2 セル目、以後 2 フレームごとに 1 セル。
- DAS チャージ (押下継続時間) は**ミノのロック・スポーンをまたいで保持**する (押しっぱなしなら新ミノが即座に壁まで走る)。
- 左右同時押しは**後から押した方**を優先。逆方向に切り替えたらチャージは 0 から。
- 壁・ブロックで移動失敗してもチャージ・タイマーは進める (離れた瞬間に流れるいわゆる壁チャージ)。

### 12.3 回転・その他入力

- A / B は押下エッジで 1 回だけ回転 (キーリピートなし)。
- 下キーのソフトドロップはリピート概念なし (押している間 `G_soft` 適用)。
- 上キーのハードドロップは押下エッジのみ。誤爆防止のため「新ミノスポーン時に上キーが押しっぱなしなら無視 (再押下が必要)」とする。

### 12.4 IRS / IHS (任意機能)

- IRS (Initial Rotation System): スポーン時に回転ボタンが押されていたら、スポーン直後に回転を 1 回適用。
- IHS (Initial Hold System): スポーン時にホールドボタンが押されていたら即ホールド。
- どちらも**実装任意**。実装しない場合は単に無視してよい。

出典: https://tetris.wiki/DAS , https://harddrop.com/wiki/DAS , https://tetris.wiki/Tetris_Guideline

## 13. トップアウト条件

以下のいずれかで即ゲームオーバー:

| 名称 | 条件 |
|---|---|
| **ブロックアウト** (Block Out) | 新規スポーン (ホールドからの再スポーン含む) 位置の 4 セルのいずれかが既存ブロックと重なる (§3.2-2。スポーン直後の 1 行落下より前に判定) |
| **ロックアウト** (Lock Out) | ミノがロックされたとき、4 セル**すべて**が可視領域の外 (`y >= 20`) にある |

- 一部でも `y <= 19` にかかってロックした場合はゲーム続行。
- ガベージ (せり上がり) は本作に存在しないため Garbage Out はなし。

出典: https://tetris.wiki/Top_out , https://tetris.wiki/Tetris_Guideline

## 14. ゲームフロー (状態遷移)

```
        ┌─────────┐  START押下(レベル選択確定)   ┌─────────┐
        │  TITLE  │ ───────────────────────────→ │ PLAYING │
        │(シード  │                               │         │──┐START
        │ 収集中) │ ←──────────────────────────  │         │←─┘(PAUSED往復)
        └─────────┘        任意ボタン             └────┬────┘
             ↑                                        │ トップアウト(§13)
             │              任意ボタン            ┌────┴────┐
             └────────────────────────────────── │GAME OVER│
                                                  └─────────┘
```

### 14.1 TITLE

- フレームカウンタでシードを収集 (§5.3)。
- 開始レベル 1〜15 を選択 (十字キー上下)。START で PLAYING へ。遷移時に PRNG をシードし、袋を 2 つ生成してネクストを満たす。

### 14.2 PLAYING (1 フレームの処理順)

決定的テストのため、毎フレーム以下の順で処理する:

1. 入力読み取り (エッジ検出込み)
2. ポーズ入力 → PAUSED へ (以降の処理をスキップ)
3. ホールド (§6.2) / ハードドロップ (§7.4) — 排他。ハードドロップならロック処理まで一気に実行
4. 回転 (§4)
5. 左右移動 (DAS/ARR, §12.2)
6. 重力 / ソフトドロップ (§7): アキュムレータ加算・落下
7. 接地判定とロックディレイ更新 (§8) → タイマー 0 なら ロック処理 (§8.3: 固定 → ロックアウト判定 → T-Spin 判定 → ライン消去 → スコア/レベル更新 → 次ミノスポーン (ブロックアウト判定))
8. 描画用状態確定 (ゴーストピース位置 = 現在位置から純落下させた位置。表示は薄色/枠のみ。ガイドライン必須機能)

- 3〜5 の同一フレーム内優先順位: ホールド > ハードドロップ > 回転 > 移動。

### 14.3 PAUSED

- 全タイマー・アキュムレータ・PRNG を停止 (状態を一切進めない)。
- フィールドを隠す必要はない (任意)。START で PLAYING に復帰。

### 14.4 GAME OVER

- スコア・ライン数・レベル・(任意) ハイスコアを表示。任意ボタンで TITLE へ。

出典: https://tetris.wiki/Tetris_Guideline (ゲームフローの必須要素), 2009 Tetris Design Guideline
